# 自动化工作流搭建

## 核心概念

超级个体的终极形态：让系统替你工作。自动化工作流 = 代码级的可靠性 + 7×24 小时的持续性。

[Vigma](https://vigma.app) 提供了超级个体自动化的基础设施，帮助你从手动操作进化到系统自运转。

## 自动化的三个层级

| 层级 | 描述 | 投入时间 | 回报周期 | 示例 |
|------|------|---------|---------|------|
| L1 任务自动化 | 单个重复任务自动执行 | 1-2 小时 | 即时 | AI 自动写周报 |
| L2 流程自动化 | 多步骤串联自动执行 | 半天-1天 | 1 周 | 内容创作→发布→推广 |
| L3 系统自动化 | 完整业务闭环自动运转 | 1-2 周 | 1 月+ | 自动获客→转化→交付 |

## 实操步骤

### Step 1：绘制你的工作流地图

```
列出你每周做的事 → 标记重复频率 → 识别自动化机会 → 计算省时收益

示例：
| 任务 | 频率 | 耗时/次 | 月耗时 | 自动化难度 | 优先级 |
|------|------|---------|--------|-----------|--------|
| 写周报 | 1/周 | 1h | 4h | 低 | ⭐⭐⭐⭐⭐ |
| 社交媒体发布 | 5/周 | 0.5h | 10h | 低 | ⭐⭐⭐⭐⭐ |
| 客户跟进邮件 | 3/周 | 0.5h | 6h | 中 | ⭐⭐⭐⭐ |
| 数据报表 | 2/周 | 1h | 8h | 中 | ⭐⭐⭐⭐ |
| 内容选题 | 1/周 | 2h | 8h | 高 | ⭐⭐⭐ |
```

### Step 2：从最高 ROI 的任务开始自动化

**自动化 #1：内容发布流水线**
```
触发：Notion 数据库新文章状态→"待发布"
  → AI 生成社交媒体文案（n8n + ChatGPT）
  → AI 生成封面图（Midjourney API）
  → 自动发布到各平台（Buffer/n8n）
  → 发送通知到飞书/Slack
```

**自动化 #2：客户管理流水线**
```
触发：新客户咨询表单提交
  → AI 评估客户质量（ChatGPT 分类）
  → 自动发送个性化回复邮件
  → 创建 Notion 客户卡片
  → 添加到跟进日历
```

**自动化 #3：数据追踪流水线**
```
触发：每天 22:00
  → 抓取各平台数据（n8n + API）
  → AI 生成数据洞察（ChatGPT）
  → 更新 Vigma 数据看板
  → 周五额外生成周报
```

### Step 3：n8n 核心节点速查

| 节点 | 用途 | 常见配置 |
|------|------|---------|
| Webhook | 接收外部触发 | URL + 认证 |
| HTTP Request | 调用 API | URL + Headers + Body |
| OpenAI | AI 处理 | Model + Prompt + Temperature |
| IF/Switch | 条件分支 | 字段比较规则 |
| Set | 数据转换 | 映射字段 |
| Cron | 定时触发 | 表达式/间隔 |

### Step 4：搭建你的自动化中台

在 [Vigma](https://vigma.app) 上构建：
- 📋 自动化工作流目录（所有工作流一览）
- 🔔 异常告警（自动化失败时通知）
- 📊 执行日志（运行记录和统计）
- 🔄 一键启停（灵活控制）

## 注意事项

- ⚠️ 先手动跑通流程，再自动化——不要自动化一个坏流程
- ⚠️ 每个自动化都要有异常处理和告警
- ⚠️ 保存自动化模板——可复用到其他场景
- ⚠️ API Key 和敏感信息用环境变量，不要硬编码
- ⚠️ 定期检查自动化是否还在正常工作
